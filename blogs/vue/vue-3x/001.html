# 计数器

Vue.createApp({ 
  data(){
    return {
      counter: 0,
      timer: null
    }
  },
  mounted() { // 页面加载完成后-自动执行
    console.log(this) // Proxy {…}
    this.timer = setInterval(()=>{
      this.counter += 1; // this.counter指向的是data中的counter
      // ---- 等同于上面的写法 ----
      // this.$data.counter += 1; 
    },1000)
  },
  unmounted(){ // 页面卸载 - 清除定时器
    this.timer && clearInterval(this.timer)
  }
  template:'<div>{{counter}}</div>' 
}).mount('#app')

# 基础api

  • 插值表达式：
    • 常规： {{}}
    • 渲染html: v-html='val'
    • 渲染一次： v-once

    • 常规：v-bind:title='val'

    • v-else-if

    • v-show

# v-if 和 v-show 的区别？

v-if更加灵活，可以增加多个判断，比如v-else-iif和else，而v-show不具备这样的灵活性。
v-show控制DOM元素显示，其实控制的是css样式，也就是display:noe。

# 普通方法和计算属性的区别 ?

  • 方法methods：只要页面重新渲染，就会重新执行方法
  • 计算属性computed: 当计算属性依赖的内容发生变更时，才会重新执行计算

# 侦听器watch和计算属性computed的区别 ?

  • 计算属性computed必须要返回一个值，而且在页面渲染的同时就会执行里边的业务逻辑，也就是会先执行一遍你写的业务逻辑。
  • watch只有发生变化时才会执行，也就是说值没有变化，它是不执行里边业务逻辑的。

# method、watch和computed三者如果都能实现相同的功能，它们之间的取舍和使用优先级 ？

  • computed 和 method都能实现的功能，建议使用computed,因为有缓存，不用渲染页面就刷新
  • computed 和 watch 都能实现的功能，建议使用 computed，因为更加简洁。

# 基本绑定事件

const app=Vue.createApp({ 
    data(){
        return{
            count:0
        }
    },
    methods:{
      addCountClick(n,event){
        this.count += n;
        console.log(event.target) // 点击元素的Dom
      },
      handleClick1(){
        alert('触发了')
      }
    },
    template:`
      <div>
        <div>数量{{count}}</div>
        <button @click='addCountClick(n,$event)'>增加</button>

        // 简介写法
        <button @click='count++'></button>

        // 同时触发多个事件【用逗号隔开】
        <button @click='addCountClick(n,$event),handleClick1()'>触发多个事件</button>
      </div>` 
    }) 
    const vm=app.mount("#app")

# 事件修饰符@click

  • stop：组织冒泡
  • prevent： 阻止默然行为【列如a,form】
  • capture: 改成捕获模式,从下到上
  • self：点击自己才执行
  • once: 事件只执行一次
  • passive: 解决滚动时性能的修饰符
